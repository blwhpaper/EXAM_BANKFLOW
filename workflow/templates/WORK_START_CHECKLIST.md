# Work Start Checklist

## Startup Checks

- `branch check`: confirm the current branch matches the intended task scope
- `git status check`: confirm the working tree is understood before new edits begin
- `required SOT files check`: read `workflow/TASK_INDEX.md`, `workflow/TASK_STATE.json`, and task-required audit/context files
- `dataset mutation boundary check`: confirm whether the task is allowed to touch datasets and record the answer before proceeding
- `expected outputs check`: list the exact files or artifacts expected from the task before making edits
